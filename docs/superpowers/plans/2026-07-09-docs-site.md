# Docs Site on GitHub Pages — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish the existing gundi-action-runner guides as a browsable MkDocs Material site on GitHub Pages, with a PR link-check gate and deploy-on-main.

**Architecture:** `mkdocs.yml` at the repo root reads the existing `docs/` tree (pages unmoved), excludes the internal `docs/superpowers/` directory, and adds one new landing page. A standalone `docs.yaml` workflow builds with `mkdocs build --strict` on PRs (link-rot gate) and deploys from `main` via the official Pages actions — no `gh-pages` branch.

**Tech Stack:** mkdocs-material ~9.5, mkdocs-exclude ~1.0 (the PADAS org's existing docs stack, per gundi-client-v2), `actions/upload-pages-artifact` + `actions/deploy-pages`.

**Spec:** `docs/superpowers/specs/2026-07-09-docs-site-design.md`.

### Task 2: Pages workflow and setup runbook

**Files:**
- Create: `.github/workflows/docs.yaml`
- Modify: `RELEASING.md` (one-time setup section)

**Interfaces:**
- Consumes: `mkdocs build --strict` (Task 1's verified command).
- Produces: deploy-on-main pipeline; Pages setup documented.

- [ ] **Step 1: Write the workflow**

`.github/workflows/docs.yaml`:

```yaml
name: Docs

on:
  pull_request:
    paths:
      - "docs/**"
      - "mkdocs.yml"
      - ".github/workflows/docs.yaml"
  push:
    branches:
      - main
    paths:
      - "docs/**"
      - "mkdocs.yml"
      - ".github/workflows/docs.yaml"

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - name: Install docs tooling
        run: python -m pip install "mkdocs-material~=9.5" "mkdocs-exclude~=1.0"
      - name: Build (strict link check)
        run: mkdocs build --strict
      - name: Upload Pages artifact
        if: github.event_name == 'push'
        uses: actions/upload-pages-artifact@v3
        with:
          path: site

  deploy:
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    permissions:
      pages: write
      id-token: write
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

(The docs job installs the two packages directly rather than `-e .[docs]` — the site build doesn't need the library, and skipping it avoids compiling pyjq on the runner. The `docs` extra exists for local contributors.)
